# Memory Synapse DB 仕様書 v1.0 — 表示・手書き情報

## 第5章 通常表示と受け皿

### 5.1 カード上部

`display_mode: source`では、大きなカード自身のカテゴリ固有情報をカード上部の主表示とする。

表示項目はカード種類ごとに次のとおりとする。

| 種類 | 主表示 |
|---|---|
| Tag | 元表記、自由メモ |
| Mention | 元の`@文字列`、名前、電話番号、Web、自由メモ |
| Location | 場所名、座標、住所、活動ID、元ファイル、人間の記憶 |

関連投稿は件数だけで終わらせず、元のWikiリンクを開けるようにする。

### 5.2 受け皿

受け皿は大きなカードの下部に表示し、先頭に大きなカード自身、その後に`members`の順で並べる。

`display_mode: source`では、大きなカード自身の受け皿内表示を次の短縮版とする。

- 種類
- カード名
- 「上に表示中」の表示
- 関連投稿数

他の構成員は、カテゴリ固有情報、`## 手書き情報`があればその内容、関連投稿リンクを通常表示する。

短縮版は画面表示だけに適用する。Markdownから大きなカード自身のカテゴリ固有情報を削除しない。

### 5.3 単独カード

`## Memory Synapse`がなく、他の大きなカードにも所属しないカードは単独カードとして表示する。受け皿は表示しない。

単独カードでも`手書き`ボタンを表示し、同じファイルへ手書き情報を保存できる。ただし、構成員が一枚だけの`memory_synapse`は作らない。単独カードに手書き情報がある場合は、手書き情報を上部、その下に元のカテゴリ固有情報を通常表示する。

## 第6章 手書き情報

### 6.1 保存場所

手書き情報は、手書きを行った個別カード自身の`## 手書き情報`へ保存する。融合状態とは独立して保持し、大きなカード変更や分離で自動削除しない。

### 6.2 YAML形式

````markdown
## 手書き情報

```yaml
memory_synapse_note:
  display_name: null
  aliases: []
  name: null
  phone: []
  web: []
  geo:
    lat: null
    lng: null
    alt: null
  address:
    full: null
    components:
      country: null
      prefecture: null
      city: null
      district: null
      street: null
      postal_code: null
  note: null
```
````

| 項目 | 型 | 初期値 | 用途 |
|---|---|---|---|
| `display_name` | 文字列または`null` | `null` | 大きなカードとして人間が付ける表示名 |
| `aliases` | 文字列配列 | `[]` | 人間が確認した別名 |
| `name` | 文字列または`null` | `null` | 人・店舗・団体等の名称 |
| `phone` | 文字列配列 | `[]` | 確認した電話番号 |
| `web` | 文字列配列 | `[]` | Web、メール、SNS等 |
| `geo.lat` | 数値または`null` | `null` | 人間が確認・補正した緯度 |
| `geo.lng` | 数値または`null` | `null` | 人間が確認・補正した経度 |
| `geo.alt` | 数値または`null` | `null` | 人間が確認・補正した高度 |
| `address.full` | 文字列または`null` | `null` | 人間が確認・補正した住所全文 |
| `address.components.*` | 文字列または`null` | `null` | 人間が確認・補正した住所内訳 |
| `note` | 文字列または`null` | `null` | 意味、記憶、判断等の自由メモ |

空文字は保存時に`null`へ変換する。複数値欄の空行は除去し、入力順を維持する。同じ値は一件にする。

### 6.3 手書き操作

1. カード内の`手書き`ボタンを押す。
2. 第6.2節の全入力欄を、現在の手書き値または初期値で同じカード内に表示する。
3. `保存`では入力内容と変更対象を表示して確認する。
4. 確認後、同じファイルの`## 手書き情報`を追加または更新する。
5. `キャンセル`ではMarkdownを変更しない。

融合中の大きなカードで保存した場合は、`display_mode`を`handwritten`へ変更する。単独カードでは表示モードを保存せず、手書き情報を常に上部へ表示する。

### 6.4 手書き表示

`display_mode: handwritten`では、`memory_synapse_note`をカード上部の主表示とする。空欄は編集画面では表示し、閲覧画面では値がある項目だけ表示する。

それまで大きなカード上部に表示していたカテゴリ固有情報は、受け皿内の大きなカード自身の通常表示へ戻す。短縮版にはしない。

`通常表示へ戻す`操作では`display_mode`だけを`source`へ変更し、手書き情報は削除しない。

### 6.5 補正情報

位置情報や住所を人間が補正する場合は、`memory_synapse_note.geo`または`memory_synapse_note.address`へ保存する。

次は変更しない。

- Locationカードの移行時点の`geo`と`address`
- 関連投稿のFrontmatter
- Instagram原本
- `output_IGC`の上流成果物

画面では「移行時点」と「手書き・補正後」を区別して表示する。
