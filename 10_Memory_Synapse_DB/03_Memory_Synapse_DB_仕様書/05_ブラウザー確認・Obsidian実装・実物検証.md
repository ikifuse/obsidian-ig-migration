# Memory Synapse DB 仕様書 v1.0 — ブラウザー確認・Obsidian実装・実物検証

## 第11章 ブラウザー確認画面

### 11.1 データ

ブラウザー確認では、実データを直接読み書きせず、次を含む検証用カードを使用する。

- Mention二枚
- Location二枚
- Tag二枚
- 各種類の関連投稿
- 手書き情報があるカードとないカード
- 二つの融合済みグループ

ページ再読込で初期状態へ戻せるようにする。確認中の操作はブラウザー内の検証状態だけへ反映する。

### 11.2 画面構成

画面は次の三領域で構成する。

1. カード一覧
2. 選択中の大きなカードと受け皿
3. 確認ポップアップまたは手書き入力欄

カード一覧では種類、カード名、単独・融合済みの状態を確認できるようにする。種類は文字だけでなく、Mention、Location、Tagを区別できる表示を付ける。

### 11.3 必須操作

- ドラッグ＆ドロップによる融合開始
- `融合へ追加`
- 大きなカードの選択
- `大きなカードを変更`
- `手書き`
- `通常表示へ戻す`
- `このカードを分離`
- `融合をすべて解体`
- `元に戻す`
- `初期状態へ戻す`

### 11.4 確認ケース

| 番号 | 操作 | 合格条件 |
|---|---|---|
| B-01 | MentionとLocationを融合 | Mentionが推奨され、別カードも選択できる |
| B-02 | LocationとTagを融合 | Locationが推奨される |
| B-03 | Tag同士を融合 | 受け入れる側が推奨され、変更できる |
| B-04 | Mention同士を融合 | 受け入れる側が推奨され、変更できる |
| B-05 | 四枚を融合 | 全四枚が一つの受け皿に表示される |
| B-06 | 大きなカード同士を融合 | 入れ子にならず全構成員が一列の受け皿になる |
| B-07 | 通常表示 | 大きなカードは上部に通常表示、受け皿では短縮表示される |
| B-08 | 手書き表示 | 手書き情報が上部、元の大きなカード情報が受け皿で通常表示される |
| B-09 | 通常構成員を分離 | 選んだ一枚だけが単独カードへ戻る |
| B-10 | 大きなカードを分離 | 次の大きなカードを選べ、元情報を失わない |
| B-11 | 一枚だけ残す | 融合状態と空の受け皿が消える |
| B-12 | 元に戻す | 直前操作前の状態へ戻る |
| B-13 | キャンセル | 状態が一切変わらない |
| B-14 | 多重所属を試す | 実行されず、理由と対象が表示される |

各ケースは、結果だけでなく、オーナーが操作の意味と影響を迷わず理解できるかを確認する。

## 第12章 Obsidianプラグイン

### 12.1 実装条件

- TypeScriptで実装する。
- macOS / iMac M4のデスクトップ版Obsidianを最終対象とする。
- ObsidianのVault読込、Markdown変更、リンク生成機能を使用する。
- Pythonサーバーを、融合・分離・手書きの必須条件にしない。
- 外部ネットワークへ自動接続しない。
- プラグイン内部だけに融合関係の正本を保存しない。

### 12.2 読み込み

起動時に対象ルート内のMarkdownを読み、次の索引をメモリ上に作る。

- カードパスからカード種類
- 大きなカードから全構成員
- 構成員から所属する大きなカード
- Wikiリンクから解決先

対象ファイルの作成、変更、改名、移動、削除を検知した場合は、影響する索引を更新する。リンク切れや不正状態を検出した場合は、自動修復せず表示する。

### 12.3 Markdown更新

Markdown更新では、管理対象見出し以外を変更しない。Frontmatter、カテゴリ固有YAML、関連投稿、未知の本文、改行を維持する。

文字コードはUTF-8（BOMなし）とする。既存ファイルの改行コードは維持する。

### 12.4 外部AI

v1.0では、外部AIへデータを送る機能を実装しない。将来追加する場合は、送信対象、目的、確認画面、送信しない選択肢を別仕様として承認する。

## 第13章 Obsidian実物検証

### 13.1 検証用データ

最初は実Vaultの複製または専用の検証Vaultを使用する。日常利用Vaultへ直接導入しない。

### 13.2 合格条件

- 第11.4節の全操作をObsidian上でも再現できる。
- MarkdownをObsidian標準画面で開き、融合関係と手書き情報を読める。
- Wikiリンクから各個別カードと関連投稿へ移動できる。
- 個別カードのフォルダーとファイルが融合前後で変わらない。
- 新しい融合専用カードが生成されない。
- カテゴリ固有情報、関連投稿、Frontmatterが融合前後で一致する。
- 手書き補正によって投稿Frontmatterが変更されない。
- 大きなカード同士を融合しても入れ子にならない。
- 分離、解体、取消後も全カードと手書き情報が残る。
- プラグイン停止中でもMarkdownから主要な関係を確認できる。
- 外部通信が発生しない。

### 13.3 不合格時

一項目でも成立しない場合は、プラグイン完成としない。仕様と実装のどちらが原因かを自動的に決めず、確認済み事実、影響、変更候補を分けて報告する。

## 第14章 対象外

- AIによる意味上の同一性の自動確定
- 人間の確認を経ない自動融合・自動分離
- 未確定候補を長期間保存する常設作業台
- Instagram以外の情報源の取込み
- IGX
- 複数の大きなカードへの同時所属
- 受け皿の入れ子
- 投稿Frontmatterへの位置情報同期
- 実データを使ったブラウザー確認
