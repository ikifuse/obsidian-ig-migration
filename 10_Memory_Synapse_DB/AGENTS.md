# AGENTS — Memory Synapse DB

## 適用範囲と上位条件

- 本文書は`10_Memory_Synapse_DB/`配下の作業に適用し、リポジトリ直下の`AGENTS.md`と併用する。
- `../01_IG移行企画書v1.0.md`をプロジェクト全体の最上位条件、`01_Memory_Synapse_DB_企画書v2.1.md`を本工程の直接の企画正本とする。
- 工程間の境界とプロジェクト共通原則は`../02_IG移行設計書/00_設計書目次.md`とそこに記載された設計6冊、Memory Synapse DB工程内部の構造と責任は専用設計が定める。
- 全体設計と専用設計の不一致を見つけた場合は、どちらかを自動的に優先せず、責任範囲、事実、影響を示して変更方針の承認を得る。

## 作業範囲と停止条件

- 閲覧や変更の対象は、対象工程の目次から選んだ分冊と必要なコード・成果物に限る。
- `10_Memory_Synapse_DB/`の外側に内容変更が必要な場合は、対象、理由、影響を示して追加承認を得る。親文書への参照更新だけを、親文書の内容変更へ広げない。
- 設計変更、仕様変更、実行コード変更、ブラウザー確認画面、書き込み可能な正式Obsidianプラグイン、実物検証を別の停止点とし、直前工程の承認を次工程の変更許可に拡大しない。
- モデレーター等へ相談するための読み取り専用仮プラグインは、正式プラグインとは区別する。仮プラグインの作成を、ブラウザー確認の合格、仕様の再承認、正式プラグイン実装の許可または完成主張へ拡大しない。
- 停止点では、変更した対象、未変更範囲、検証結果、未確定事項を報告し、オーナーの確認を待つ。

## 実装環境とデータ保護

- ブラウザー確認は仕様の検証用データだけを使い、実際のVaultと`output_IGC`を書き換えない。
- ブラウザー確認前に相談用の仮プラグインを作る場合は、対象カードの読み取りと計測に限定し、Markdownの作成、変更、移動、削除を実装しない。
- 書き込み可能な正式Obsidianプラグインは、ブラウザー確認と必要な仕様変更の再承認後にTypeScriptで実装する。Pythonサーバーを融合・分離・手書きの必須条件にしない。
- `node_modules/`は再生成可能なローカル開発用依存物であり、読み込み、編集、Git管理の対象にしない。
- 最終対象はmacOS / iMac M4のデスクトップ版Obsidianとする。最初の実物検証では、実Vaultの複製または専用の検証Vaultを使い、日常利用Vaultへ直接導入しない。
- v1.0では外部AIへデータを送る機能を実装せず、外部ネットワークへ自動接続しない。
