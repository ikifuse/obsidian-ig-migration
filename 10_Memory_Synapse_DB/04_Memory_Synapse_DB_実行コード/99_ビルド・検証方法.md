# Memory Synapse DB ビルド・検証方法

## 1. 初回準備

ターミナルで本フォルダーへ移動し、必要な開発用パッケージを導入する。

```bash
cd 10_Memory_Synapse_DB/04_Memory_Synapse_DB_実行コード
npm install
```

## 2. 型検査

```bash
npm run check
```

TypeScriptの型と参照先が成立しているかを確認する。ファイルが生成されない検査である。

## 3. 自動テスト

```bash
npm test
```

融合、推奨、分離、多重所属検出、対象パス判定を検証する。

## 4. 生成

```bash
npm run build
```

この一回のビルドで次を生成する。

```text
../05_Memory_Synapse_DB_ブラウザー確認版/
├── index.html
├── app.js
└── styles.css

../06_Memory_Synapse_DB_仮プラグイン/
└── memory-synapse-db/
    ├── main.js
    ├── manifest.json
    └── styles.css
```

## 5. ブラウザー確認

`05_Memory_Synapse_DB_ブラウザー確認版/index.html`をブラウザーで開く。検証用カードだけを使用し、実Vaultと`output_IGC`を読み書きしない。

## 6. 仮プラグイン確認

> [!warning] 日常利用Vaultへ入れない
> 最初は専用の検証Vaultまたは実Vaultの複製を使用する。

`06_Memory_Synapse_DB_仮プラグイン/memory-synapse-db/`フォルダーを、検証Vaultの`.obsidian/plugins/`へフォルダーごと配置する。

現段階のプラグインは対象カードとWikiリンクの読取・計測だけを行い、Markdownを書き換えない。
