# Memory Synapse DB 実行コード目次

## 1. このフォルダーの役割

ここには、Memory Synapse DBのブラウザー確認版とObsidianプラグインを生成するTypeScriptの元コード、生成用素材、ビルド設定を置く。

人間が読むTypeScript本体は`TypeScript元コード/`へまとめ、役割が重ならない次の5つに分ける。番号は処理の流れを示すもので、別々のアプリケーションを意味しない。

| 番号 | フォルダー | 役割 |
|---|---|---|
| 01 | [01_データ構造](TypeScript元コード/01_データ構造/00_このフォルダーの役割.md) | カード、融合状態、手書き情報、操作結果の形を定める |
| 02 | [02_操作処理](TypeScript元コード/02_操作処理/00_このフォルダーの役割.md) | 融合、分離、変更、検証、取消の規則を実行する |
| 03 | [03_データ入出力](TypeScript元コード/03_データ入出力/00_このフォルダーの役割.md) | ブラウザー内データとObsidian Vaultの読み取りを受け持つ |
| 04 | [04_画面](TypeScript元コード/04_画面/00_このフォルダーの役割.md) | ブラウザーとObsidianの画面を表示し、人間の操作を受け取る |
| 05 | [05_共通処理](TypeScript元コード/05_共通処理/00_このフォルダーの役割.md) | 設定、入力値整理、エラー、共通型を受け持つ |

## 2. 処理の流れ

```text
TypeScript元コード/
├── 01_データ構造
│       ↓
├── 02_操作処理 ← 03_データ入出力
│       ↓
├── 04_画面
│       ↑
└── 05_共通処理
```

画面は融合規則を独自判断せず、`02_操作処理`を呼び出す。データの保存先は操作処理へ直接埋め込まず、`03_データ入出力`でブラウザー用とObsidian用を分ける。

## 3. 生成物

この実行コードをビルドすると、次の二つを生成する。

- `../05_Memory_Synapse_DB_ブラウザー確認版/`
- `../06_Memory_Synapse_DB_仮プラグイン/memory-synapse-db/`

生成物だけを修正せず、変更は原則として本フォルダーのTypeScriptまたは素材へ行い、再ビルドする。
