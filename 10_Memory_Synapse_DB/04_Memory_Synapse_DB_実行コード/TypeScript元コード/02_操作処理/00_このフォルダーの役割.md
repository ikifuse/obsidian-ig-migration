# 02_操作処理の役割

## 担当すること

- 融合候補の展開と大きなカードの推奨
- 融合グループ同士を入れ子にせず一列に統合
- 大きなカードの変更、構成員の分離、融合全体の解体
- 手書き情報の保存、通常表示と手書き表示の切替
- 多重所属などの状態検証
- ブラウザー確認中の直前操作の取消

## 担当しないこと

- HTMLやObsidian画面の表示
- Markdownファイルの直接書き換え
- 外部通信

## 含まれるコード

`融合.ts`、`分離.ts`、`大きなカード変更.ts`、`手書き保存.ts`、`表示切替.ts`、`状態参照.ts`、`状態検証.ts`、`元に戻す.ts`に責任を分ける。

`操作処理.test.ts`は、推奨、平坦な融合、一枚残し、多重所属検出を自動確認する。
