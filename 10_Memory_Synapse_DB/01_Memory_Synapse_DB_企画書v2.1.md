# Memory Synapse DB 企画書 v2.1

## 第1章 プロジェクトの位置付け

### 1.1 目的

Memory Synapse DBは、Instagramから移行した投稿、ハッシュタグ、メンション、位置情報などを、オーナーが長期的に育てられる知識へ変えるための基盤である。

単なる投稿閲覧、バックアップ、検索画面を目的としない。人生の記録を人間とAIが共通して理解し、振り返り、関係を見つけ、将来の判断や創造へ活用できる「第二の脳」の一部を構築する。

### 1.2 Instagram移行プロジェクトとの関係

本企画書は、`01_IG移行企画書v1.0.md`を最上位条件とするMemory Synapse DB工程の企画を定める。

IGP・IGR・IGSを検証・完走し、IGCを実行・検証して完成させた`output_IGC`を、本工程への確定入力として受け取る。`output_IGC`は移行・加工後の成果物であり、Instagramから取得した原本そのものとは区別する。

IGXは現在の完走条件およびブロッカーにしない。IGXを将来も完全に廃止するかどうかは、本企画では決定しない。

### 1.3 現在の承認済み完走点

現在の完走点は、検証済みの`output_IGC`をObsidianへ導入し、Memory Synapse DBのObsidianプラグインを動作させ、オーナーが実データを用いて次の成果をObsidian上で確認できる状態までとする。

- 個別カードと関連投稿を確認できる
- 人間の判断で個別カードを融合・分離できる
- 融合後も個別カード固有の情報を確認できる
- 必要な個別カードだけを分離して戻せる
- 人間が意味や記憶を追記し、大きなカードを育てられる
- 誤操作の対象と影響を確認し、情報を失わずに戻せる

IGP・IGR・IGS、IGC、プラグイン、Obsidian上での成果確認を分断せず、現在承認されている完走点まで到達する。

## 第2章 解決したい課題

### 2.1 情報の散在

投稿本文、写真、位置情報、ハッシュタグ、メンション、人物や場所に関する記憶は、異なるファイルや情報源に分かれている。個別情報を保存するだけでは、同じ人物、場所、経験、意味のつながりを理解しにくい。

### 2.2 表記と意味の違い

同じ人物や場所でも、時期、入力方法、言語、表記揺れによって別のカードとして現れる。一方、似た表記でも意味が異なる場合があるため、システムだけで同一性を確定すると誤った融合が起きる。

完全一致による機械的な重複整理と、人間が意味を確認して行う知識の融合は区別する。

### 2.3 人間が扱える規模を超える情報量

数千件の投稿や多数の個別カードを、ファイル一覧と手作業だけで継続的に整理することは難しい。全体を見渡すことと、必要な一件を詳しく確認することの両方を、非エンジニアであるオーナーが無理なく行える必要がある。

### 2.4 知識を育てる場所の不足

Instagramから抽出された情報だけでは、オーナーの記憶、意味、判断理由、現在の理解を十分に表せない。事実として受け取った情報を失わず、人間が後から意味を加え、関係を見直し、知識として育てる場所が必要である。

## 第3章 実現したい価値

### 3.1 個別カードを知識へ育てる

ハッシュタグ、メンション、位置情報などの個別カードを、抽出結果のまま保存するだけで終わらせない。人間が意味や記憶を加え、関連するカードと結び付けることで、長期的に使える知識へ育てる。

### 3.2 不要なカードを増やさない

融合管理だけを目的とするカードを自動的に増やさない。既存の個別カードが他の個別カードを受け入れ、人間の手書き情報も加えながら、一枚の大きなカードへ成長することを基本とする。

人間が手書き情報を加える場合も、別のカードを自動生成せず、現在の大きなカードを育てる。

### 3.3 個別情報を失わない融合

融合後も、融合に参加した各個別カードの種類、元の表記、固有情報、関連投稿を区別して確認できる状態を保つ。情報を平らに混ぜたり、一方へ置き換えたりすることを融合の前提にしない。

大きなカードでは、融合した個別カードの情報を一か所から確認できるようにする。同じ情報を不必要に重複表示せず、人間が全体と個別の両方を理解できる状態を目指す。

### 3.4 可逆的な分離

融合した個別カードは、必要な一枚だけを分離して独立した状態へ戻せるようにする。大きなカードに選ばれた個別カードを分離する場合も、残った個別カードから次の大きなカードを選び、情報を失わずに関係を組み直せることを目指す。

### 3.5 人間が大きなカードを決める

システムは、カードの種類や操作状況に基づいて大きなカードの候補を提示できる。ただし、候補は提案であり、融合後または分離後にどの個別カードを大きなカードとするかは人間が最終決定する。

### 3.6 人間とAIが共通して理解できる

人間が画面から意味と関係を理解できるだけでなく、AIもMarkdownに残された情報と関係を読み取り、オーナー固有の文脈を理解するために活用できる状態を目指す。

## 第4章 データを守る原則

### 4.1 原本と加工成果物を区別する

Instagramから取得した原本は、将来再解析できるよう保持する。`output_IGC`と、そこから利用する投稿・個別カードは、原本から作成された加工成果物として区別する。

Memory Synapse DBの操作によって、上流工程の原本を変更・消失させない。

### 4.2 事実と人間が育てる知識を区別する

投稿や個別カードが移行時点で持っていた情報と、人間が後から追記した意味、記憶、判断、関係を区別できる状態を保つ。

後から確認・補正した情報をどこへ保存し、画面でどのように見せるかは、原本保持と区別可能性を損なわないことを条件として設計工程で定める。

### 4.3 個別カードを破壊しない

融合・分離・手書きによって、個別カード固有の情報や関連投稿へのつながりを失わせない。大きなカードが変わっても、融合に参加した個別カードの情報を確認できる状態を維持する。

### 4.4 人間が読める形で残す

知識と関係は、特定のプラグインだけが理解できる内部状態へ閉じ込めない。プラグインが利用できない場合も、MarkdownとObsidianのリンクから、人間とAIが主要な情報と関係を確認できる状態を目指す。

## 第5章 現在の対象範囲

### 5.1 対象

- 検証済みの`output_IGC`にある投稿、個別カード、関連投稿へのつながり
- ハッシュタグ、メンション、位置情報の個別カード
- 個別カードおよび大きなカードの確認
- 人間の判断による融合、追加、分離
- 大きなカードの選択と変更
- 大きなカードへの手書き情報の追記と編集
- 融合後の全個別情報と関連投稿の確認
- 操作前の確認と、情報を失わずに戻せる安全性
- Obsidian上で日常的に利用するためのプラグイン

### 5.2 現在の対象外

- システムまたはAIによる意味上の同一性の自動確定
- 人間の確認を経ない自動融合・自動分離
- 未確定候補を長期間保存する常設の作業台
- 融合管理だけを目的とするカードの自動生成
- Instagram以外の情報源を今回の完走条件へ追加すること
- IGXを現在の完走条件またはブロッカーにすること

常設の作業台は、将来、未確定候補の保存・比較が具体的に必要になった場合に、目的と対象を改めて確認して検討する。

## 第6章 継続して守る原則

### 6.1 人間が最終判断を行う

AIとシステムは候補提示、比較、整理、実装支援を行う。採用、融合、分離、意味付け、大きなカードの選択は、人間が内容を確認して最終決定する。

### 6.2 オーナーが理解し続けられる

技術的な高度さよりも、非エンジニアであるオーナーが結果と影響を理解し、長期間にわたり管理・改善・運用できることを優先する。

### 6.3 ローカルを知識の正本とする

知識の正本は、オーナーが管理するiMac内のローカルデータとして保持する。プラグインや外部サービスが、オーナーの確認なしに個人データを外部へ送信してはならない。

外部AIの利用自体は一律に禁止しない。利用する場合は、対象、目的、送信する情報をオーナーが確認して判断する。

### 6.4 特定のAIやプラグインに依存しない

現在のAI、特定ベンダー、Memory Synapse DBプラグインだけに依存しない。将来別のAIやアプリを利用する場合も、同じ知識を継続して活用できる状態を目指す。

### 6.5 長期的に育て続ける

一度完成して終了するシステムではなく、人生の記録、人間の理解、AIの進化に合わせて知識を育て続けられる基盤とする。

## 第7章 将来構想

### 7.1 マルチソース対応

将来はInstagramだけでなく、Facebook、GPS履歴、YouTube、資格学習、各種メモ、ライフログ、その他の情報源を同じ知識基盤へ統合する。

現在のInstagram移行とMemory Synapse DBは、その長期的な知識基盤を成立させる最初の工程として位置付ける。

### 7.2 AIによる支援

AIが投稿や個別カードから、関連候補、表記揺れ、意味付け、検索候補を提示し、オーナー固有の文脈を理解した支援へ発展させる。AIは候補を提示しても、人間の確認なしに知識の関係を確定しない。

### 7.3 知識ネットワークの成長

人物、場所、タグ、出来事、作品、学習、行動履歴などの関係を可視化し、人間とAIが人生のつながりを多面的に理解できる知識ネットワークへ育てる。
