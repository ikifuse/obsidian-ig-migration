# Memory Synapse DB 実装・検証ワークフロー

## 役割と開始条件

- 実装と成果物検証は、承認済み仕様をコードと実物で成立させる工程である。仕様の意味を実装便宜で変更しない。
- オーナーが対象実装と停止点を承認していることを開始条件とする。
- 実行コード変更、ブラウザー確認画面、書き込み可能な正式Obsidianプラグイン、Obsidian実物検証を別の停止点として扱う。
- モデレーター等への相談用に読み取り専用仮プラグインを先に生成する場合は、正式プラグインと区別し、実装工程の完了またはブラウザー確認の代替としない。

## 共通の読み込み

1. `../03_Memory_Synapse_DB_仕様書/00_仕様書目次.md`を読む。
2. `../04_Memory_Synapse_DB_実行コード/00_実行コード目次.md`を読む。
3. 今回の操作・データ・検証に必要な仕様分冊と、`../04_Memory_Synapse_DB_実行コード/TypeScript元コード/`内の対象責任フォルダーにある`00_このフォルダーの役割.md`だけを選ぶ。
4. 対象コードと対象成果物を読む。`node_modules/`は読まない。
5. 設計との不一致判断が必要な場合だけ、`../02_Memory_Synapse_DB_設計書/00_設計書目次.md`から対応分冊を読む。

## 実行コードと生成物

1. 元コード、ビルド設定、ブラウザー素材、プラグイン素材は`../04_Memory_Synapse_DB_実行コード/`で管理する。
2. 人間が読むTypeScript本体は`../04_Memory_Synapse_DB_実行コード/TypeScript元コード/`へまとめ、`01_データ構造`、`02_操作処理`、`03_データ入出力`、`04_画面`、`05_共通処理`の5責任に分ける。
3. `npm run build`で、`../05_Memory_Synapse_DB_ブラウザー確認版/`と`../06_Memory_Synapse_DB_仮プラグイン/memory-synapse-db/`を生成する。
4. `05`と`06`の生成物だけを直接修正しない。修正は`04`のTypeScriptまたは素材へ行い、型検査、自動テスト、再ビルドを実行する。
5. `node_modules/`は`npm install`で再生成し、Git管理しない。

## ブラウザー確認画面

1. `../05_Memory_Synapse_DB_ブラウザー確認版/`を使用する。
2. 仕様の検証用カードだけを使い、実際のVaultと`output_IGC`を読み書きしない。
3. ページ再読込で初期状態へ戻せる独立した確認環境とする。
4. `../03_Memory_Synapse_DB_仕様書/05_ブラウザー確認・Obsidian実装・実物検証.md`の確認ケースと、対象操作の仕様分冊を検証する。
5. オーナーの確認後、仕様変更が必要な場合は実装を停止し、仕様工程へ戻す。

## 相談用の読み取り専用仮プラグイン

1. `../06_Memory_Synapse_DB_仮プラグイン/memory-synapse-db/`を、モデレーター等へ実物を示して技術的意見を求めるための叩き台とする。
2. 対象カードの読み取り、Wikiリンク数、読取時間等の計測と表示に限定する。
3. Markdownの作成、変更、移動、削除、融合状態の保存、操作履歴の保存を実装しない。
4. 外部通信を行わず、専用の検証Vaultまたは実Vaultの複製だけで確認する。
5. 仮プラグインが動作しても、仕様の合格、正式プラグインの完成または日常利用可能とは扱わない。

## 書き込み可能な正式Obsidianプラグイン

1. ブラウザー確認と、必要な仕様変更の再承認後に開始する。
2. TypeScriptで実装し、Pythonサーバーを融合・分離・手書きの必須条件にしない。
3. プラグイン内部データだけを融合関係の正本にせず、管理対象外のMarkdown内容を変更しない。
4. 外部ネットワークへ自動接続せず、外部AIへデータを送る機能をv1.0に含めない。

## Obsidian実物検証

1. macOS / iMac M4のデスクトップ版Obsidianを最終対象とする。
2. 実Vaultの複製または専用の検証Vaultを使い、日常利用Vaultへ直接導入しない。
3. 仕様の全合格条件を実物で確認し、一項目でも成立しない場合は完成としない。
4. 不合格時は仕様不備か実装不具合かを自動的に決めず、事実、影響、変更候補を分けて報告する。

## 停止点

- 各実装・検証段階で、変更ファイル、未変更範囲、実行した検証、合否、未確定事項を報告する。
- ブラウザー確認画面の完成を、Obsidianプラグイン実装の許可またはMemory Synapse DB全体の完成に拡大しない。
- 読み取り専用仮プラグインの生成を、書き込み可能な正式プラグインの実装許可または完成に拡大しない。
