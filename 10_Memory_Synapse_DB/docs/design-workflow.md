# Memory Synapse DB 設計ワークフロー

## 役割

- `01_Memory_Synapse_DB_企画書v2.1.md`の承認済み内容を、Memory Synapse DB工程内部の構造、責任、データ間の関係、守る条件へ分解する。
- 工程間の境界とプロジェクト共通原則は親設計に従い、専用設計から変更しない。
- 本ワークフローは進め方だけを定め、設計本文を置き換えない。

## 読み込み

1. `../02_Memory_Synapse_DB_設計書/00_設計書目次.md`を読む。
2. 目次の読み分けから、今回の判断に必要な設計分冊だけを選ぶ。
3. 企画条件の確認が必要な場合は`../01_Memory_Synapse_DB_企画書v2.1.md`の対象箇所を読む。
4. IGP・IGR・IGS・IGCとの境界に影響する場合だけ、`../../02_IG移行設計書/00_設計書目次.md`から対応分冊を選ぶ。

## 判断と反映

1. 今回確定する設計範囲と、変更しない親側の境界を明示する。
2. 確認済み事実、仮定、提案、未確認事項を分ける。
3. データ保護、人間の最終判断、可逆性、失敗時の影響を確認する。
4. 仕様が推測なしに定義できる設計判断だけを、対応する現役分冊へ統合する。

具体的な項目名、型、保存位置、画面配置、ボタン、確認文言、処理順、判定式は仕様で定める。

## 停止点

- 設計変更後は、変更分冊、維持した分冊、仕様・コード・成果物への影響、未確定事項を報告する。
- オーナーが設計を確認し、仕様工程への移行を承認するまで仕様を変更しない。
