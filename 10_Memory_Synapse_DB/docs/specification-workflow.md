# Memory Synapse DB 仕様ワークフロー

## 役割と開始条件

- 専用設計で確定した内容を、ブラウザー確認画面とObsidianプラグインへ推測なしに実装できる規則へ変換する。
- オーナーが対応設計を確認し、仕様工程への移行を承認していることを開始条件とする。
- 本ワークフローは進め方だけを定め、仕様本文を置き換えない。

## 読み込み

1. `../03_Memory_Synapse_DB_仕様書/00_仕様書目次.md`を読む。
2. 目次の読み分けから、今回の対象仕様分冊だけを選ぶ。
3. `../02_Memory_Synapse_DB_設計書/00_設計書目次.md`から、対応する設計分冊だけを読む。
4. コード・成果物は、比較が必要な対象範囲だけを読む。

`../../04_IG移行仕様書/06_Memory_Synapse_DB_仕様書v1.2.md`は比較資料であり、現役仕様または専用設計へ対応済みとは推測しない。

## 仕様化と比較

1. 対象設計の意味、責任境界、禁止事項、成立条件を維持する。
2. 入出力、項目名、型、初期値、保存先、処理順、判定式、画面操作を必要な範囲で確定する。
3. 現行コード・成果物との不一致は、「維持」「微調整」「設計変更候補」「実装不具合」「未確認」に分類する。
4. 設計の矛盾、欠落、不成立を発見した場合は仕様化を停止し、設計工程へ戻す。

## 停止点

- 仕様変更後は、変更分冊、維持した分冊、コード・成果物への影響、未確定事項を報告する。
- オーナーが仕様を確認し、対象とする実装工程への移行を承認するまでコードを変更しない。
