# Memory Synapse DB 設計書 v1.0 — 表示・手書き・分離

## 第5章 表示と手書きによる成長

### 5.1 通常表示

通常表示では、選ばれた大きなカードの情報をカード上部の主表示として見せる。受け皿にも大きなカード自身が含まれることを示すが、同じ情報を二重に長く表示しないため、大きなカード自身の受け皿内表示は短縮できる。

短縮は表示上の扱いであり、大きなカード自身の元情報を削除、変更、別形式へ置換する処理ではない。他の構成員は、それぞれの固有情報と関連投稿を通常の単位で確認できるようにする。

### 5.2 手書き表示

手書きは、現在の大きなカード内にある操作入口から開始する。別の融合カードや手書き専用カードを自動生成しない。

手書きを選ぶと、現在の大きなカード上部に、人間が意味、記憶、名称等を入力できる空欄を表示する。入力された手書き情報を大きなカードの主表示とし、それまで主表示だった個別カードの情報は、受け皿の中で短縮せず通常表示へ戻す。

この切替えによって、大きなカードとして選ばれた個別カードの地位や元情報が消えるわけではない。大きなカードのファイル自体を育てながら、融合に参加した全個別カードの情報を受け皿で確認できる状態を保つ。

### 5.3 手書き情報の保護

手書き情報は、人間が育てた知識として、移行時点の情報と区別して保持する。大きなカードの変更、分離、融合全体の解体によって保持先が変わる場合は、人間の確認なしに削除、分配、上書きしない。

具体的な保持場所、編集単位、空欄、表示切替え方法は仕様書で定める。

## 第6章 分離と組み直し

### 6.1 個別カードの分離

融合に参加したどの個別カードも、その一枚だけを分離できる。分離したカードは独立した個別カードとして戻り、元情報と関連投稿を保持する。

大きなカードではない構成員を分離する場合、残る構成員と大きなカードの関係は維持する。

### 6.2 大きなカード自身の分離

現在の大きなカードに選ばれている個別カードも分離対象にできる。誤って大きなカードにした場合だけに限定せず、人間が関係を見直す通常操作として扱う。

大きなカード自身を分離する場合は、残る構成員から次の大きなカードを人間が選ぶ。システムは融合時と同じ基本順序で候補を提示できるが、自動確定しない。

### 6.3 構成員が一枚になる場合

分離後に構成員が一枚だけ残る場合、融合状態を終了し、そのカードを通常の個別カードとして扱う。融合管理だけを目的とする空の受け皿を残さない。

### 6.4 全体の解体と取消

一枚だけを外す分離、現在の大きなカードを外す分離、融合全体の解体、直前操作の取消を区別する。

融合、分離、大きなカード変更は、複数の関係と手書き情報へ影響し得る一操作として扱う。途中で一部だけが変更された状態を成功とせず、操作前へ戻すか、安全に再開できる単位を設ける。
