# Memory Synapse DB 設計書 v1.0 — 大きなカード・受け皿・融合

## 第3章 大きなカードと受け皿

### 3.1 大きなカード

大きなカードは、融合専用に新しく作るカードではない。融合に参加する既存の個別カードから、人間が一枚を選び、そのカード自体を大きなカードとして育てる。

融合管理だけを目的とする新しいMarkdownファイルを自動生成しない。人間が手書き情報を加える場合も、新しいカードを作らず、現在の大きなカードを成長させる。

### 3.2 受け皿

受け皿は、大きなカードを含む、融合に参加した全個別カードの関係を表す。受け皿は別のカードや常設作業台ではなく、大きなカードから確認できる融合済みメンバーのまとまりである。

大きなカード自身も受け皿の構成員として扱う。ただし、同じファイルから自分自身へWikiリンクを張ることは必須としない。他の個別カードへのWikiリンクと、人間が読める表示によって全構成員を確認できればよい。

受け皿では、各個別カードの種類、元表記、固有情報、関連投稿を個別の単位として保持する。異なるカードの情報を区別できない一つの文章や項目へ平らに混ぜない。

### 3.3 常設作業台との区別

融合を確定する前の候補と確認は、一時的な画面またはポップアップで扱う。確定済みの融合関係を保持する受け皿とは区別する。

未確定候補を長期間保存・比較する常設作業台は現在設置しない。将来、具体的な必要性が確認された場合に別工程として検討する。取消や復旧に必要な履歴は、常設作業台とは別の保全情報として扱う。

## 第4章 融合

### 4.1 人間による確定

システムは融合候補と大きなカード候補を提示できるが、意味が同じか、同じ人物・場所・対象として育てるかを自動確定しない。融合の実行と大きなカードの選択は、人間が内容と影響を確認して決定する。

Tag、Mention、Location等のカテゴリが同じ場合も異なる場合も、人間が同じ知識として育てると判断したカードを融合できる。参加枚数を二枚に限定しない。

### 4.2 大きなカードの候補

大きなカードの候補は、次の順序を基本の推奨とする。

1. Mention
2. Location
3. Tag

Mentionを含む場合はMentionを第一候補とする。同じカテゴリ同士では、ドラッグ＆ドロップ等の操作で受け入れる側を候補とする。これは初期提案であり、人間は確認時に別の個別カードを選べる。

具体的な選択画面、操作方法、候補が複数ある場合の並べ方は、ブラウザー上で確認した後に仕様書で定める。

### 4.3 大きなカード同士の融合

大きなカード同士を融合する場合も、融合に参加していた全個別カードを一つの受け皿へ並べ直す。受け皿の中に別の受け皿を残す入れ子構造にはしない。

融合後の大きなカードは、基本の推奨を保ちながら人間が選ぶ。融合前に大きなカードだった個別カードも、選ばれなければ通常の構成員として保持する。
