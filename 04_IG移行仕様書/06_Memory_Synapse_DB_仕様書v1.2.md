# 06_Memory_Synapse_DB_仕様書v1.2

※本書は「06_Memory_Synapse_DB_仕様書v1.1.md」を前提に、Synapseメモを人間が後から育てる受け皿として整理したv1.2仕様書である。

共通仕様については、一律で **`01_IG移行共通仕様書v1.2.md`** を適用する。

---

## 1. v1.2の目的

v1.2では、ハッシュタグ・メンション・位置情報を自動でまとめない。

抽出されたタグ名・メンション名・場所情報は、原則として個別のSynapseメモまたは確認用レコードとして生成する。

Synapseメモは、投稿を見るための一覧ではなく、Obsidian内でリンク構造を作り、後から意味や連絡先や場所情報を追記するための受け皿として扱う。

ただし、メンションと場所では、Instagram元データから確定できる事実の単位が異なる。

メンションは、キャプションから確認できる元の `@文字列` を抽出事実とする。元の `@文字列` だけを根拠に、実在するInstagramアカウント、現在の所有者、本人性を確定しない。

場所情報は投稿に付いたチェックイン1回を基本単位とし、場所名・座標・住所だけで自動統合しない。

---

## 2. 対象メモ

対象は以下の3種類とする。

* タグメモ：`Instagram_Logs/Synapses/Tags/`
* メンションメモ：`Instagram_Logs/Synapses/Mentions/`
* 場所メモ：`Instagram_Logs/Synapses/Locations/`

ここでいうメモは、Obsidianで扱う1つのMarkdownファイルを指す。

メンションメモは、抽出工程が保持した元の `@文字列` から接続する。表記または出典の統合によって実在アカウントが同一だと自動確定せず、Obsidian上では同じ `[[Wikiリンク]]` 先を一つのカードとして扱う。

場所メモは、場所名テキストの種類数を実在場所数として扱わない。同じ場所名であっても、投稿に付いたチェックインが別スポット・別座標・別住所を指す可能性があるため、生成時点では同一場所として確定統合しない。

---

## 3. 自動揺らぎ判定をしない

v1.2では、以下を行わない。

* ハッシュタグの自動揺らぎ判定
* メンションの自動揺らぎ判定
* 位置情報の自動揺らぎ判定
* 自動統合
* 自動分離

同じものに見える名前であっても、生成時点では勝手に1つへまとめない。

特に場所情報では、場所名・座標・住所を自動統合の根拠にしない。

場所名一覧は確認用索引として扱い、同名の場所名が同一スポットであるとは判断しない。

将来、統合・分離・リンクON/OFFを行う場合は、Memory Synapse DBのObsidianプラグインまたは作業UIで、人間が確認して操作する。

---

## 4. 個別Synapseメモに持たせない情報

個別のタグメモ・メンションメモ・場所メモには、以下の集計値を持たせない。

* 出現回数
* 投稿回数

関連する投稿メモへの接続は、各Synapseメモの `## 関連投稿` にWikiリンクとして保持する。

---

## 5. SystemLogsの扱い

SystemLogsは、静的な確認用ログとして扱う。

SystemLogsに一覧を作る場合でも、それは人間が育てる本体ではない。

人間が追記して育てる本体は、`Instagram_Logs/Synapses/` 配下の個別Synapseメモである。

---

## 6. タグメモ

タグメモには、元のハッシュタグ表記と、後から自由に書けるメモ欄を置く。

ハッシュタグは、その場の気分、時代ごとの投稿癖、大文字小文字の揺れ、雑に付けたタグなどを多く含むため、抽出時に自動の揺らぎ判定を行わない。

タグメモには、出現回数、投稿回数を持たせない。関連する投稿メモへの接続は `## 関連投稿` にWikiリンクとして保持する。

ハッシュタグ固有の情報欄へ、リンクON/OFF、統合、分離、操作履歴、判断メモを重複して設置しない。これらの共通作業台は、設計で項目が確定した後にIGCが設置する。

````markdown
## 管理

- [x] 採用

## 情報

```yaml
hashtag_note:                                 # ハッシュタグを人間が育てる欄
  hashtag: "#SampleTagA"                         # 投稿から抽出されたハッシュタグ表記
  note: null                                  # 自由メモ
```

## 関連投稿

[[2024-01-06-12-00-00_IG_0006]]
[[2020-08-22-15-35-22_IG_1535]]
[[2020-08-22-16-08-30_IG_1534]]
````

ハッシュタグメモには、投稿ごとの `hashtag_extraction`、`source_post`、`source_raw`、`raw` を繰り返し出力しない。`IGP_HASHTAG_EXTRACTION:(投稿ID):START` および `IGP_HASHTAG_EXTRACTION:(投稿ID):END` の生成管理印も出力しない。元のハッシュタグ表記と出典は投稿本文、投稿メモ、個別RawDataに保持する。

ハッシュタグメモの出力形式は [[01_IG移行共通仕様書v1.2#4.2.1 ハッシュタグの情報欄]] を正とする。

## 7. メンションメモ

メンションメモには、分類チェックを置かない。チェックボックスは、採用ON/OFFのための1つだけとする。

メンションメモには、人、店舗、団体等の情報を人間が後から育てる `mention_note` を1件だけ置く。その下の `## 関連投稿` に、そのメンションが含まれる投稿メモへのWikiリンクを並べる。

````markdown
## 管理

- [x] 採用

## 情報

```yaml
mention_note:                                 # 人・店舗・団体などを育てる欄
  mention: "@sample_account_001"                   # 投稿から抽出された@表記
  name: null                                  # 実際の名前を人間が記入する欄
  phone: []                                   # 確認した電話番号を追加する欄
  web:                                        # メール・Webサイト・SNSなど
    - "https://www.instagram.com/sample_account_001/"
                                               # @表記から生成したInstagram URL
  note: null                                  # 自由メモ
```

## 関連投稿

[[2022-05-11-17-20-55_IG_836]]
[[2022-05-24-16-48-12_IG_833]]
[[2022-05-26-18-03-07_IG_829]]
````

`phone`（電話番号）と `web`（Web・連絡先）は、確認した値を後から追加できる複数値とする。独立した `email`（メールアドレス）欄および `related_locations`（関連場所）は設置しない。生成したInstagram URLは元の `@文字列` に対応する参照先であり、アカウントの現在の存在、現在の所有者、本人性を保証しない。

同一投稿内に完全に同じ `@文字列` が複数回ある場合、繰り返しは投稿本文とRawDataに保持する。同じ `[[Wikiリンク]]` 先はObsidian上で一つのカードとして扱い、出現回数ごとのメモまたはカードを生成しない。

メンションメモには、投稿ごとの `mention_extraction`、`source_post`、`source_raw`、`raw` を繰り返し出力しない。`IGP_MENTION_EXTRACTION:(投稿ID):START` および `IGP_MENTION_EXTRACTION:(投稿ID):END` の生成管理印も出力しない。元の `@文字列` と出典は投稿本文、投稿メモ、個別RawDataに保持する。

メンションメモの出力形式は [[01_IG移行共通仕様書v1.2#4.2.2 メンションの情報欄]] を正とする。

## 8. 場所メモ

場所メモには、`location_note`、座標、住所、活動ID、活動の元ファイル、人間の記憶を1件のYAMLとして置く。その下の `## 関連投稿` に、その場所情報を持つ投稿メモへのWikiリンクを並べる。

````markdown

## 管理

- [x] 採用

## 情報

```yaml
location_note:
  location: "サンプル地点A"    # Instagramから抽出された場所名

geo:                               # 緯度・経度の座標
  lat: 35.000001              # 緯度
  lng: 135.000001             # 経度
  alt: null                        # 高度

address:                           # 住所
  full: null                       # 住所全文
  components:                     # 住所の内訳
    country: null                  # 国
    prefecture: null               # 都道府県
    city: null                     # 市区町村
    district: null                 # 地区
    street: null                   # 番地・通り
    postal_code: null              # 郵便番号

activity_id: "activity_000001"     # 活動をまとめる番号

source_files:                      # 活動の元ファイル
  - "[[activity_000001.gpx]]" # GPXファイル
  - "[[activity_000001.kml]]" # KMLファイル
  - "[[activity_000001.tcx]]" # TCXファイル
  - "[[activity_000001.csv]]" # CSVファイル

note: null                         # 元ファイルから分からない人間の記憶
```

## 関連投稿

[[2022-09-18-14-53-08_IG_641]]
````

場所メモには、投稿ごとの位置観測YAMLを繰り返し出力しない。`IGP_LOCATION_OBSERVATION:(投稿ID):START` および `IGP_LOCATION_OBSERVATION:(投稿ID):END` の生成管理印も出力しない。元の場所名、チェックイン情報、出典は投稿メモと個別RawDataに保持する。

場所メモの出力形式は [[02_IGP移行仕様書v1.2#2.2.1 Location Synapseメモ]] を正とする。

## 9. IGCとの接続と共通作業台

Memory Synapse DB の Python サーバー実行ファイルは、`09_Memory_Synapse_DB/09_Memory_Synapse_DB_実行.py` とする。

IGCは、既存のカテゴリ別情報、人間が育てる空欄、元投稿および元データへのリンクを削除・置換しない。

ハッシュタグ・メンションの独立した抽出ID、融合後の大きなカード専用ID、融合グループIDは必須にしない。Obsidian上では、元メモへの `[[Wikiリンク]]` を識別と接続の中心とする。

全カテゴリに共通する作業台、融合、分離、取消、復元、判断履歴および操作履歴の具体的な項目とYAMLは、現在の設計では未確定である。以前の共通識別情報と作業台の仮YAMLを確定仕様として使用しない。IGC設計のサンドバックが成立した後に、本仕様書へ追加する。

IGCは意味統合・分離・投稿リンク書換えを実行しない。同じカテゴリ内の同じファイル名かつ同じ内容だけを完全重複として整理し、同名で内容が異なる場合は停止する。カテゴリをまたぐ同名も自動統合しない。

カテゴリ固有のタグ情報、メンション情報、チェックイン・位置候補・GPX・KML等の場所情報は維持する。将来、共通作業台を追加する場合も、カテゴリ固有情報を理由なく削除してはならない。

## 10. 将来工程として扱うこと

以下の人間操作はv1.2の移行工程およびIGCでは実行しない。

* チェック操作による投稿メモ内リンクの削除・復帰
* 統合操作
* 分離操作
* 場所名・座標・住所を使った自動統合
* Undo/Redo
* 操作ログつきのカードUI

v1.2では、これらを将来実装できるように、個別Synapseメモの受け皿を先に整える。
