# 01_目的からゴールへの流れ v1.0

この図は、`obsidian-ig-migration` の本流だけを一本で見るためのものです。
詳細に枝分かれせず、「何を目指し、どの順番で進み、どこで手動判断が入るか」を掴むことを優先します。

```mermaid
flowchart LR
    A["目的<br/>Instagram / Meta の長年の記録を<br/>Obsidian 上の第二の脳にする"]
    B["01_企画<br/>何を目指すか<br/>目的<br/>優先順位"]
    C["02_設計<br/>構造<br/>責務分離<br/>設計判断"]
    D["03_仕様<br/>実装できるルール<br/>入力 / 出力<br/>処理の定義"]
    E["実行プログラム群<br/>04_IGP<br/>05_IGR<br/>06_IGS<br/>07_IGX（保留）<br/><br/>※ IGXだけ保留<br/>ここを今深掘りすると<br/>本流の整理が止まるため後回し"]
    F["成果物確認<br/>output_IGP<br/>output_IGR<br/>output_IGS<br/>output_IGX（保留）<br/>output_IGC<br/><br/>※ output_IGXだけ保留<br/>扱いは今後整理"]
    G["手動投入<br/>あなたが順番を見ながら<br/>成果物を Obsidian へ写す"]
    H["ゴール<br/>Obsidian 上の第二の脳として<br/>使える状態にする"]

    A --> B --> C --> D --> E --> F --> G --> H
```

## 読み方

- `01` から `03` までは、文書を整える段階です。
- `04_IGP` / `05_IGR` / `06_IGS` は本流の個別実行です。
- `07_IGX` は存在するが、今は本流整理を止めないために保留しています。
- `output_IGC` は本番側の統合成果物です。
- 最後は人間が投入順序を見ながら Obsidian へ反映します。
