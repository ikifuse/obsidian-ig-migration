# マインドマップ企画書

このフォルダは、`obsidian-ig-migration` を理解しやすくするための、最小構成のマインドマップを置く専用フォルダである。

## まず全体理解したい人へ

NotebookLM の理解用マインドマップは、公開範囲を確認した共有リンクだけを必要時に案内する。

この資料は、全体像を頭で掴むための入口として使う。
一方で、このフォルダ内の `.canvas` と `md` は、Obsidian 管理と GitHub 共有のために残す。

目的は、オーナー本人と GitHub を見に来た人が、

* プロジェクト全体像
* 文書と実行の関係
* 再開時にどこから読むか

を、固い文章を全部読まなくても掴めるようにすること。

## 運用方針

* マインドマップは増やしすぎない
* 基本の地図は `01_目的からゴールへの流れ` と `02_全体像` の2系統に絞る
* `03_引き継ぎ_現時点の問題整理` の運用は一旦保留する
* 開発期間が空いても、引き継ぎ書や引き継ぎマインドマップを必ず作るとは限らない
* 引き継ぎマインドマップを再開する場合は、曖昧な引き継ぎ書だけを根拠にせず、現行の企画書・設計書・仕様書とオーナー確認に基づいて作る
* 完全自動追従は今はやらない
* Obsidian で見てすぐ分かることを最優先にする
* `.canvas` は Obsidian で編集しやすい JSON Canvas 形式、`md` は GitHub 共有用として併用する

## このフォルダの見方

* NotebookLM: 理解用
* `.canvas`: Obsidian での編集・調整用。JSON Canvas 対応ツールでも扱える場合がある
* `md`: GitHub 上でそのまま読める共有用

## 今の判断

`03_引き継ぎ_現時点の問題整理` は、開発期間が空いた場合の再開支援として作成していた。

しかし、引き継ぎ書が常に必要とは限らず、曖昧な引き継ぎ内容を現在の正しい判断として残すと、後続作業が誤った構想へ進む危険があることが分かった。

そのため、引き継ぎマインドマップの運用は一旦保留する。必要性が生じた場合に、現行の正式資料とオーナー確認から内容を作れることを確認したうえで再開する。

旧 `03_引き継ぎ_現時点の問題整理` は、当時の経緯を確認するための完了済み参考資料として保管する。
