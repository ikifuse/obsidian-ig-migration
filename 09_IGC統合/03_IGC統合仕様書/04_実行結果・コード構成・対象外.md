# IGC統合仕様書 v1.0 — 実行結果・コード構成・対象外

## 11. 実行結果

### 11.1 正常時

正常時は終了コード0とし、標準出力へ一つのJSONオブジェクトを出力する。

JSONは少なくとも次を持つ。

| 項目 | 型 | 内容 |
|---|---|---|
| `mode` | string | `dry-run`または`replace` |
| `ok` | boolean | 全検証の成否 |
| `replaced` | boolean | 恒久出力を置換したか |
| `inputs` | object | 3入力系統のパス |
| `categories` | object | カテゴリ別結果 |
| `categories.*.source_card_counts` | object | 系統別カード件数 |
| `categories.*.source_list_counts` | object | 系統別一覧件数 |
| `categories.*.input_union_count` | integer | 完全一致和集合件数 |
| `categories.*.multi_source_item_count` | integer | 複数系統に存在した統合キー数 |
| `categories.*.output_count` | integer | 出力カード件数 |
| `categories.*.related_post_count` | integer | 出力関連投稿リンク総数 |
| `categories.*.validation` | object | 検証項目ごとのboolean |

### 11.2 異常時

異常時は終了コード1とし、標準出力へ一つのJSONオブジェクトを出力する。

正常時の項目に加えて`error`オブジェクトを置き、少なくとも`type`、`phase`、`message`を持たせる。特定できる場合は`category`、`source`、`display_name`、`path`、`post_link`を含める。

人間向け表記または投稿リンクを報告する場合も、エラー箇所の確認に必要な対象だけに限定し、一覧全体を標準出力へ複製しない。

## 12. 実行コード構成

`04_IGC統合実行`は次の責任分割を維持する。

| ファイル | 責務 |
|---|---|
| `01_IGC_メイン実行v1_0.py` | 入出力名、カテゴリ、物理名規則、全処理の呼び出し、ドライラン、JSON結果、終了コード |
| `02_IGC_入力解析v1_0.py` | カードと一覧の解析、入力系統ごとの相互照合 |
| `03_IGC_データ統合v1_0.py` | 完全一致統合、関連投稿加算、物理名決定、Synapse生成、統合後一覧の生成 |
| `04_IGC_品質検証v1_0.py` | 入力、準備出力、対象外保護の検証 |
| `05_IGC_安全置換v1_0.py` | 対象6か所のバックアップ、置換、失敗時復元 |

## 13. 今回実装しないもの

* 大文字小文字、別名、類似名、座標等による自動名寄せ
* 同一カテゴリ内の意味融合
* カテゴリ横断融合
* UUID、融合後カードID、融合グループID、所有者の設定
* 大きなメモ、共通作業台、共有メモ、判断履歴
* 投稿リンクの書き換え
* 統合プレビューUI
* 分離・取消・復元UI
* Memory Synapse DBの設計・仕様・コード変更

これらをIGCコードへ先回りして実装しない。
