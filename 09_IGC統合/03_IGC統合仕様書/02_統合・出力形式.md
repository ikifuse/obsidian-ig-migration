# IGC統合仕様書 v1.0 — 統合・出力形式

## 5. 完全一致統合

### 5.1 統合キー

統合キーは、次の二要素の組とする。

```text
(カテゴリキー, 人間向け表記)
```

人間向け表記はPython文字列の完全一致で比較する。大文字小文字、全角半角、Unicode正規化形、先頭記号、句読点その他の文字が一文字でも異なれば別の統合キーとする。

例えば`#ABC`と`#abc`、`#abc`と`＃abc`、`大阪城`と`大阪城天守閣`は別カードとして出力する。カテゴリが異なる同一文字列も別カードとする。

### 5.2 共通情報の一致

同じ統合キーが複数系統に存在する場合、3.3で分けた共通情報を比較する。

改行をLFへ統一した後の共通情報が文字単位で完全一致する場合だけ、一組を出力する。比較ではFrontmatter、見出し、カテゴリ固有YAML、人間が育てる欄、コメント、空白、その他の本文を省略しない。出力には、2.1の入力順で最初に存在するカードの共通情報を使用する。

共通情報が異なる場合は、最初の系統を採用せず、内容を混ぜず、全処理を異常終了する。

### 5.3 関連投稿の加算

同じ統合キーに属する関連投稿リンクを、次の順番で一つの列へ加える。

1. `output_IGP`カード内の順番
2. `output_IGR`カード内の順番
3. `output_IGS`カード内の順番

投稿リンクの意味、日時、画像、出来事を比較してまとめない。リンク文字列を変更しない。

同じ統合キーについて、複数系統に完全に同じ投稿リンクがある場合は、重複除去せず入力異常として全処理を停止する。カードと対応一覧に同じリンクが一回ずつ記載されることは正常な相互表現であり、重複とは数えない。

### 5.4 一系統だけに存在するカード

一系統だけに存在する統合キーも出力する。共通情報と関連投稿は同じ規則で再構成し、推測した情報や他系統の空欄を追加しない。

### 5.5 出力カード形式

出力カードは、共通情報を一組置き、その後に次の形式で統合後の関連投稿を置く。

```markdown
（入力と完全一致する共通情報）

## 関連投稿

[[IGPの投稿ID]]
[[IGRの投稿ID]]
[[IGSの投稿ID]]
```

ファイル末尾はLF一つで終了する。

IGCは次を追加または変更しない。

* `synapse_id`
* `synapse_facets`
* `integration_state`
* `integration_target_id`
* `integration_source_ids`
* `igc_sources`
* UUID
* 融合ID、融合グループID、所有者
* IGC統合・分離作業台
* 融合・分離・取消・復元・判断履歴の欄

## 6. 出力物理ファイル名

### 6.1 基底名

人間向け表記から、物理ファイル名の基底名を次の順序で作る。

1. Tagsだけ、先頭の`#`または`＃`を一文字除く。MentionsとLocationsは先頭記号を除かない。
2. `\ / : * ? " < > |`を一文字ごとに`_`へ置換する。
3. CRとLFを空白へ置換する。
4. 前後の空白を除く。
5. 先頭から100文字に制限する。
6. 空文字になった場合は`unnamed`とする。

この処理は物理保存名を作るためだけに行い、人間向け表記と統合キーは変更しない。

### 6.2 macOS上の物理名衝突

カテゴリごとに、基底名をUnicode NFDへ変換して`casefold()`した値を、macOS上の物理名衝突判定キーとする。

異なる人間向け表記が同じ衝突判定キーを持つ場合、そのグループに属する全カードの基底名へ次の接尾辞を付ける。

```text
--{12桁の小文字16進数}
```

12桁は次のUTF-8文字列のSHA-256から先頭12桁を取得する。

```text
{カテゴリ性質値}\0{人間向け表記}
```

カテゴリ性質値は次に固定する。

| カテゴリ | カテゴリ性質値 |
|---|---|
| Tags | `Synapse/Tag` |
| Mentions | `Synapse/Mention` |
| Locations | `Synapse/Location` |

例えば`#ABC`と`#abc`は統合せず、別々の決定的な物理名で同じTagsディレクトリへ保存する。

接尾辞付与後の物理名もNFD＋`casefold()`で一意でなければ異常終了する。接尾辞は物理ファイル衝突を避けるための保存名であり、Synapse IDまたは意味融合IDとしてカード本文へ書き込まない。

### 6.3 出力ファイル

出力ファイルは次の場所へ保存する。

```text
output_IGC/Instagram_Logs/Synapses/{カテゴリディレクトリ}/{物理名}.md
```

人間向け表記、Frontmatterの`aliases`、H1、カテゴリ固有値は、安全名に置き換えない。

## 7. SystemLogs出力

### 7.1 項目集合

IGCの一覧には、対応カテゴリの出力Synapseカードを一項目ずつ載せる。

一覧の人間向け表記集合とカードの人間向け表記集合は完全一致しなければならない。入力3一覧を単純連結せず、5.1の統合キーごとに一項目を出力する。

### 7.2 出力形式

カテゴリごとの見出しを次に固定する。

| カテゴリ | 見出し |
|---|---|
| Tags | `# ハッシュタグ一覧 - IGC統合` |
| Mentions | `# メンション一覧 - IGC統合` |
| Locations | `# 場所一覧 - IGC統合` |

一覧本文は次の形式とする。

```markdown
# 一覧名 - IGC統合

初期状態で全て採用（`- [x]`）です。除外したい項目は `- [x]` を `- [ ]` にしてください。

---

- [x] [[Instagram_Logs/Synapses/カテゴリ/物理名|人間向け表記]]
出現回数: N回

出現投稿:
  - [[投稿ID]]
---
```

出力時のチェック状態は全項目`- [x]`とする。

`N`は、統合後カードの関連投稿リンク数とする。一覧の出現投稿は、統合後カードと同じ値・同じ順番で出力する。

### 7.3 項目順

一覧項目は、次の順番で安定して出力する。

1. 出現回数の降順
2. 同数の場合は、人間向け表記をPython文字列として比較した昇順

この順番は意味、優先度、融合候補を表さない。
