# IGC再設計時の判断材料

> [!warning] この文書の位置付け
> この文書は、IGCの確定設計・仕様ではない。
> 前工程の設計中に判明した、IGC再設計時にサンドバックする必要がある判断材料を残す。
> IGC設計段階で内容を確認し、成立した判断だけをIGC設計書末尾の付箋へ記録する。

## メンションと位置情報の関係

### 背景

現在のIGC設計書では、Mentionの専門情報として「関連場所」が挙げられている。

一方、前工程のメンション設計を確認した結果、投稿本文にメンションと位置情報が同時に存在しても、それだけでは人・店舗・団体と場所の関係は確定できないことが分かった。

### 前工程が確実に渡せる事実

- 投稿本文に特定の `@文字列` が存在した
- 投稿に位置観測が存在した
- 両方が同じ投稿を出典としている

前工程では、「その人や店舗がその場所に関係している」とは確定しない。

### IGC再設計時に検討する案

メンション固有の出力に「関連場所」を持たせず、メンションと位置観測が同じ投稿を出典としている事実を判断材料にする。

人・店舗・団体と場所の関係は、IGC以降で人間が確認してから設置する。

### この段階で確定しないこと

- IGCで関係をどのような形式で保持するか
- 関連場所をメンションメモ内に表示するか
- 共通の融合用作業台で扱うか
- 同じ投稿に存在したものを候補として表示するか
- Memory Synapse DBでの融合・分離方法

これらはIGC設計段階で、最新の前工程設計と位置情報仕様を確認してからサンドバックする。

### IGC設計時の確認事項

1. 最新のIGP・IGR・IGS設計で、メンションと位置情報が何を事実として渡すか確認する
2. 既存IGC設計書の「Mentions: アカウント、関連場所、連絡先、Web、自由メモ」を再検討する
3. 同じ投稿に存在することと、意味上の関係があることを区別する
4. 人間が確定していない関係を自動で成立させない
5. サンドバックが成立した判断だけをIGC設計書末尾へ付箋として残す

## 文書の終了条件

この判断材料がIGC設計書の付箋へ反映された後、この文書は役割を終える。
反映先を末尾に記録してから、`完了済み参考資料` へ移動する。
